Changes from 0.9.4 to 0.9.5

Due to several projects we had to work on in 2006, more than half a year passed since the last release. Release 0.9.5 brings a lot of new features that became needed in this time. The major changes are:

1. Documentation.
The whole documentation has been updated. This opened the oppurtunity to revalidate some part of SUMO and make them so available to the public in a documented way. Two of those "found" features are vehicle rerouter and emitter. Both are also mentioned below.
For letting you what has changed quickly, the changed parts of the documentation are marked. Furthermore, documentation is now also available as .pdf-files.
The documentation may be found at http://sumo.sourceforge.net/docs/documentation.shtml

2. Junction-internal vehicle movements.
Normally in SUMO vehicles jump over junctions. Since this release there is a support for simulating traffic as it takes place in real life - vehicles are entering a junction area and are running smoothly over the junctions.
This feature is not yet used by default. You have to force the building of junction-internal lane information within NETCONVERT/NETGEN using the option --add-internal-links. This is already done for all the networks within the examples. You also have to force the simulation to use this information and simulate vehicle movement over junctions. This is done by passing the option --use-internal-lanes to the simulation (SUMO/GUISIM). Due to this there are two versions of configuraion files supplied for each of the examples, one for the normal version, one that used internal lanes.
As you may already see by the option names, this feature is not yet completely stable.
Further information on building networks that contain inner-junction lanes may be found at 4.4.9. Adding inner-junction Traffic.

3. Vehicle classes.
SUMO and the other applications from the package can now handle different "vehicle classes". Currently, these vehicle classes can only be used to disallow the usage of certain lanes or streets.
Further information may be found at 6.3.5. Vehicle Classes

4. A better support for geocoordinates.
The GUI now shows information about the cursor's position both in cartesian and geocoordinates. APIS allow the retrieval of a vehicle's position in geocoordinates (although there's no SUMO-internal usage of these APIs, yet).
The usage of the libraries "GDAL" and "PROJ.4" allows a scientific usage of geo-projected networks.
Further information may be found at 4.4.8. Converting from Geocoordinates.

5. Visualisation of geometric objects in GUISIM.
Yes, you can now make your simulations more pretty by adding water or other polygons or points of interest. Still, they are only visualised, there is no further functionality behind them.
Further information may be found at 7.4. Additional Geometry Files.

6. Vehicle emitter.
Vehicle emitter can be placed on a lane. They put vehicles exactly into the network at the place they are located at. The type of vehicle, its speed during emission and an exact emission time may be given explicitely for each vehicle to emit what allows using induction loop values from real world to generate realistic flows.
Further information may be found at 6.2.1. Emitter.

7. Vehicle rerouter.
Vehicle rerouter can be placed on a street or a set of streets and give vehicles passing those streets a new route. Several possibilities for rerouting may be used.
Further information may be found at 6.3.4. Rerouter.

!!! Warning !!!
There are also some things which are unfinished and should be handled with care. The major issues are:

  • The DFROUTER is a new and promising tool; still, it is completely unverified
  • One has to do some tweaks in order to get GDAL/PROJ.4 work; a description and patches will follow
  • The GUI (GUISIM) is currently in work, so several things may not work. Furthermore the documentation is outdated
Please commit the issues you find to the Bug List.




Other changes relevant for users:
  • All
    • added projection information to the network
  • NETCONVERT
    • xml-types are now read prior to other formats (other XML-description afterwards); This allows to define edge properties before and own connections after a foreign network has been imported
    • xml-edge-types may now store the edge function
    • VISUM: lane connections for unsplitted lanes implemented, further refactoring
  • GUI
    • shape layers added
    • added blinker visualisation and vehicle tracking (unfinished)
    • cartesian and geocoordinates are shown; changed the APIs for this
    • tls tracker now support switches between logics
  • OD2TRIPS
    • complete rework
  • Simulation
    • Rerouter verified and described
    • Emitter verified and described
    • tls switches added
  • Router
    • Speeded up the dua-computation
    • added the option to not emit vehicles from flows using a fix frequency (undocumented, yet)



Other changes relevant for developers:
  • Simulation
    • "Action" removed - was the same as Command
    • changed the Event-interface: execute now gets the current simulation time, event handlers are non-static
    • .icc-files removed
    • vehicle actors added (undocumented, yet)
    • removed the unneded "slow lane changer"
    • retrieval of microsim ids is now also done using getID() instead of id()
    • rerouter: patched the distribution usage; added possibility o load predefined routes
    • began with the extraction of the car-following-model from MSVehicle
    • added the possibility to stretch/change the current phase and consecutive phases
    • made MSVehicleControl completely responsible for vehicle handling - MSVehicle has no longer a static dictionary
  • Utils
    • made access to positions in Position2DVector c++ compliant
  • All
    • removed build number information
