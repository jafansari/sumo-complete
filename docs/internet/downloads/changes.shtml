Changes from 0.9.2 to 0.9.3

User-relevant changes

  • GUI
    • cursor position output added
    • new visualization settings implemented
    • decals added (but should be reworked and documented)
    • debugged using tables
    • renamed some mispelled output
  • Simulation
    • added a minimum simulation speed definition before the simulation ends (unfinished)
    • viewport dialog debugged
    • renamed some mispelled output
    • microsim outputs patched
      • in emissions output: renamed attribute "id" to "time"
      • in tripinfo output: renamed attribute "id" to "vehicle_id"
    • debugged variable speed signs
    • patched the documentation on raw dump (was named output within the documentation) (was [sumo-Bugs-1180985])
    • bus stops implemented
  • Netconvert
    • removed warning if no types are defined (was [sumo-Bugs-1369190])
    • network geometry reworked (unfinished)
    • plain-output now saves edge geometries (if longer than only the from/to node-positions) (was [sumo-Bugs-1369193])
    • split-geometry option added (unfinished)
    • debugged setting new connections over user-given
    • added several new features to the ArcView importer (see documentation)
    • some work on CELL-import
    • new Elmar-format adapted
    • XML-import
      • import of shapes debugged
      • node-type import debugged
    • --type-file renamed to --xml-type-files
    • arcview-import now requires the file name begin only
    • switched from --append-turnarounds to --no-turnarounds
    • switched from --no-node-removal to --remove-geometry
  • DUARouter
    • debugged vehicle type processing
    • debugged colors processing
  • All
    • errors are now printed to stderr (was [sumo-Bugs-1373537])
    • reworked error handling (were [sumo-Bugs-1371531] and [sumo-Bugs-1373540])
  • Documentation
    • complete recheck of user-docs for NETCONVERT/NETGEN/DUAROUTER
    • updated all man-pages
    • updated all help-pages
    • many examples described and added
Developer-relevant changes
  • All
    • added support for enabling explicite geometry checking under Linux
    • geometry computation now uses constants for lane width
    • final try/catch is now only used in the release version (to be reworked)
  • Simulation
    • debugged setting globals in false order
